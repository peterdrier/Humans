> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `Provider` / `ProviderKey` / `IsGoogle` to `UserEmail` and route every Google-Workspace-side reader through `IsGoogle` (and every Auth-side OAuth-link reader through `Provider`/`ProviderKey`). Wire OAuth-callback rename detection. Remove the public C# surface for `UserEmail.IsOAuth`, `UserEmail.DisplayOrder`, `User.GoogleEmail`, and `User.GetGoogleServiceEmail()` while leaving the underlying DB columns in place via EF shadow-property declarations. Backfill is a one-shot admin button. **No DB column drops in this PR** — drops across the entire 6-PR sequence are aggregated into a single late PR (referred to here as **PR 7**) that ships only after the whole sequence is verified end-to-end in production.

Also folds in three follow-ups surfaced during the upstream-promotion review of PR 2 (`peterdrier#361` → `nobodies-collective#608`):
- Doc sweep cleaning up stale `/Contacts` references that survived PR 2's surface deletion.
- `ApplicationDecisionService.ApproveAsync` / `RejectAsync` empty-email regression — `IUserService.GetByIdAsync` returns a `User` without `.Include(u => u.UserEmails)`, so post-PR1 users have `user.Email == null` and decision emails go to `""` (silently swallowed).
- Removal of PR 1's `/Admin/BackfillUserEmails` page — the operator has run it on prod; it's no longer needed.

The `User.GetGoogleServiceEmail()` doc-comment misleading-`?? Email`-fallback concern from the same review dissolves naturally because PR 3 deletes the method entirely (Task 10).

**Architecture:** PR 3 of the 6-PR email-identity-decoupling sequence. PR 1 stopped Identity-column writes and added the admin-side UserEmail-row backfill. PR 2 added virtual property overrides on `User`, swept reads to `UserEmails`, and removed the `/Contacts` admin surface. PR 3 adds the new `UserEmail` columns, retires the legacy `IsOAuth`/`DisplayOrder`/`GoogleEmail` C# surface, and turns on rename detection.

**Tech Stack:** ASP.NET Core 10, ASP.NET Identity, EF Core 10 + PostgreSQL, NodaTime (`Instant`), AwesomeAssertions, xUnit, NSubstitute, Mono.Cecil-based architecture tests.

**Spec:** `docs/superpowers/specs/2026-04-27-email-and-oauth-decoupling-design.md` (PR 3 section, revised 2026-04-30 to drop both DB unique constraints in favor of service enforcement, fix the `GoogleAdminService` translation note, and replace `b.Ignore` with shadow properties).

**Issues replaced:** `nobodies-collective/Humans#505` and `nobodies-collective/Humans#507` — but only the additive portion. The DB column drops referenced in those issues are aggregated into PR 7 of this sequence (`docs/superpowers/plans/2026-04-30-email-oauth-pr7-drop-legacy-columns.md`), which ships only after the full PR-2-through-PR-6 sequence is verified end-to-end in production. **Risk:** **Medium.** No column drops, no DB unique constraints, additive migration only. The principal failure modes are (a) auto-generated migration accidentally producing a `DropColumn` for `IsOAuth`/`DisplayOrder`/`GoogleEmail` if the shadow-property declarations are wrong (the migration reviewer agent is the gate), (b) backfill mismapping a `Provider`/`ProviderKey` pair (idempotent admin button — re-runnable), and (c) deletion-guard regression if the deletion-guard's `IsOAuth` reference (PR 1 replaced it with a counted-rows check; verify no relapse) is reintroduced anywhere.

## Hard rules in effect (non-negotiable)

- **No hard-storage drops.** No `migrationBuilder.DropColumn`, no `DropIndex`, no destructive SQL. Code-only deletion of C# properties is fine. `architecture_no_drops_until_prod_verified`.
- **No DB-enforced intra-row invariants.** Service is the contract. No `CreateIndex` for `(Provider, ProviderKey)` uniqueness or `IsGoogle` partial uniqueness. `feedback_db_enforcement_minimal`.
- **No hand-edited migrations.** 100% auto-generated by `dotnet ef migrations add`. No `migrationBuilder.Sql`. Backfills go in admin buttons. `architecture_no_hand_edited_migrations`.
- **No startup guards.** App must always start.
- **No concurrency tokens.** No `[ConcurrencyCheck]`, no row versioning.
- **No invented fields.** Add only `Provider`, `ProviderKey`, `IsGoogle`. No timestamps, status enums, derived-state surrogates. If a helper field seems useful, ASK — don't bake it in.
- **`gh` discipline.** Always `--repo peterdrier/Humans` or `--repo nobodies-collective/Humans`. Always `comments` + `author` in `--json`. Never act on a non-Peter-authored issue without confirmation.

```csharp
using AwesomeAssertions;
using Humans.Domain.Entities;
using Humans.Testing;

namespace Humans.Application.Tests.Architecture;

/// <summary>
/// Architecture test enforcing the email-identity-decoupling spec PR 3
/// (<c>docs/superpowers/specs/2026-04-27-email-and-oauth-decoupling-design.md</c>).
///
/// PR 3 deletes the public C# properties <c>UserEmail.IsOAuth</c>,
/// <c>UserEmail.DisplayOrder</c>, <c>User.GoogleEmail</c>, and the helper
/// <c>User.GetGoogleServiceEmail()</c>. The underlying DB columns are kept on
/// disk via EF shadow-property declarations and only read by the one-shot
/// backfill admin button via <c>EF.Property&lt;T&gt;(...)</c>. Any remaining
/// IL reference to the deleted setters/getters or helper inside
/// <c>Humans.Application</c> or <c>Humans.Web</c> is a build that should not
/// have shipped.
///
/// Exemption strategy: zero type-level exemptions in scanned assemblies. The
/// backfill service reads via <c>EF.Property&lt;T&gt;</c>, which never lowers
/// to a property setter/getter call, so it sits naturally outside the scan.
/// </summary>
public class UserEmailLegacyFieldRestrictionsTests
{
    private static readonly string[] ForbiddenMembers =
    {
        // UserEmail
        "set_IsOAuth", "get_IsOAuth",
        "set_DisplayOrder", "get_DisplayOrder",
        // User
        "set_GoogleEmail", "get_GoogleEmail",
        "GetGoogleServiceEmail",
    };

    private static readonly string[] ScannedAssemblies =
    {
        "Humans.Application",
        "Humans.Web",
    };

    [HumansFact]
    public void NoApplicationOrWebCode_ReferencesDeletedUserEmailOrUserLegacyMembers()
    {
        var offenders = new List<string>();

        foreach (var assemblyName in ScannedAssemblies)
        {
            var assemblyPath = ResolveAssemblyPath(assemblyName);
            using var module = ModuleDefinition.ReadModule(assemblyPath);

            foreach (var type in module.Types.SelectMany(Flatten))
            {
                foreach (var method in type.Methods.Where(m => m.HasBody))
                {
                    foreach (var instr in method.Body.Instructions)
                    {
                        if (instr.OpCode != OpCodes.Callvirt && instr.OpCode != OpCodes.Call)
                            continue;

                        if (instr.Operand is not MethodReference mref)
                            continue;

                        if (!ForbiddenMembers.Contains(mref.Name, StringComparer.Ordinal))
                            continue;

                        if (!IsUserOrUserEmail(mref.DeclaringType))
                            continue;

                        offenders.Add($"{type.FullName}.{method.Name} -> {mref.DeclaringType.Name}.{mref.Name}");
                    }
                }
            }
        }

        offenders.Should().BeEmpty(
            because: "PR 3 of the email-identity-decoupling spec deletes the public C# surface " +
                     "for UserEmail.IsOAuth, UserEmail.DisplayOrder, User.GoogleEmail, and " +
                     "User.GetGoogleServiceEmail(). Auth-side reads should use Provider != null; " +
                     "Google-Workspace-side reads should use the IsGoogle-flagged UserEmail row. " +
                     "The DB columns survive as EF shadow properties for the backfill admin " +
                     "button only. Offenders found: {0}", string.Join("; ", offenders));
    }

    private static IEnumerable<TypeDefinition> Flatten(TypeDefinition t) =>
        new[] { t }.Concat(t.NestedTypes.SelectMany(Flatten));

    private static bool IsUserOrUserEmail(TypeReference t)
    {
        var current = t;
        var guard = 0;
        while (current is not null && guard++ < 16)
        {
            if (string.Equals(current.FullName, typeof(User).FullName, StringComparison.Ordinal))
                return true;
            if (string.Equals(current.FullName, typeof(UserEmail).FullName, StringComparison.Ordinal))
                return true;
            if (current.Name.StartsWith("IdentityUser", StringComparison.Ordinal))
                return true;

            try
            {
                current = current.Resolve()?.BaseType;
            }
            catch
            {
                return false;
            }
        }
        return false;
    }

    private static string ResolveAssemblyPath(string assemblyName)
    {
        var hostDir = Path.GetDirectoryName(typeof(UserEmailLegacyFieldRestrictionsTests).Assembly.Location)!;
        var path = Path.Combine(hostDir, $"{assemblyName}.dll");
        if (!File.Exists(path))
            throw new FileNotFoundException($"Could not locate {assemblyName}.dll at {path}");
        return path;
    }
}
```

```csharp
public async Task<Result> SetProviderAsync(
    Guid userEmailId,
    string provider,
    string providerKey,
    CancellationToken cancellationToken = default)
{
    await using var tx = await _dbContext.Database.BeginTransactionAsync(cancellationToken);

    var target = await _dbContext.Set<UserEmail>()
        .FirstOrDefaultAsync(e => e.Id == userEmailId, cancellationToken);

    if (target is null)
        return Result.Failure("UserEmail row not found.");

    var conflicts = await _dbContext.Set<UserEmail>()
        .Where(e => e.Id != target.Id && e.Provider == provider && e.ProviderKey == providerKey)
        .ToListAsync(cancellationToken);

    var now = _clock.GetCurrentInstant();
    foreach (var conflict in conflicts)
    {
        conflict.Provider = null;
        conflict.ProviderKey = null;
        conflict.UpdatedAt = now;
    }

    target.Provider = provider;
    target.ProviderKey = providerKey;
    target.UpdatedAt = now;

    await _dbContext.SaveChangesAsync(cancellationToken);
    await _auditLog.WriteAsync(/* entity: target.UserId, kind: "UserEmail.SetProvider", details: ... */);
    await tx.CommitAsync(cancellationToken);

    return Result.Success();
}

public Task<UserEmail?> FindByProviderKeyAsync(
    string provider,
    string providerKey,
    CancellationToken cancellationToken = default)
{
    return _dbContext.Set<UserEmail>()
        .FirstOrDefaultAsync(
            e => e.Provider == provider && e.ProviderKey == providerKey,
            cancellationToken);
}
```

```csharp
public sealed class UserEmailProviderBackfillService : IUserEmailProviderBackfillService
{
    private readonly HumansDbContext _dbContext;
    private readonly IClock _clock;
    private readonly IAuditLogService _auditLog;
    private readonly ILogger<UserEmailProviderBackfillService> _logger;

    public UserEmailProviderBackfillService(
        HumansDbContext dbContext,
        IClock clock,
        IAuditLogService auditLog,
        ILogger<UserEmailProviderBackfillService> logger)
    {
        _dbContext = dbContext;
        _clock = clock;
        _auditLog = auditLog;
        _logger = logger;
    }

    public async Task<UserEmailProviderBackfillResult> RunAsync(CancellationToken cancellationToken = default)
    {
        var users = await _dbContext.Users
            .Include(u => u.UserEmails)
            .ToListAsync(cancellationToken);

        var logins = await _dbContext.UserLogins
            .ToListAsync(cancellationToken);

        var loginsByUser = logins.ToLookup(l => l.UserId);
        var warnings = new List<string>();
        var usersProcessed = 0;
        var providerRowsUpdated = 0;
        var isGoogleRowsUpdated = 0;
        var ambiguousMatchesWarned = 0;
        var now = _clock.GetCurrentInstant();

        foreach (var user in users)
        {
            usersProcessed++;
            var legacyGoogleEmail = _dbContext.Entry(user).Property<string?>("GoogleEmail").CurrentValue;
            var userLogins = loginsByUser[user.Id].ToList();

            // Provider / ProviderKey backfill: one row per AspNetUserLogins entry.
            foreach (var login in userLogins)
            {
                var match = ResolveProviderTargetRow(user, legacyGoogleEmail, userLogins, login, warnings, ref ambiguousMatchesWarned);
                if (match is null) continue;

                if (match.Provider == login.LoginProvider && match.ProviderKey == login.ProviderKey)
                    continue;

                match.Provider = login.LoginProvider;
                match.ProviderKey = login.ProviderKey;
                match.UpdatedAt = now;
                providerRowsUpdated++;

                await _auditLog.WriteAsync(/* user.Id, "UserEmail.Backfill.Provider", ... */);
            }

            // IsGoogle backfill: prefer the row matching legacyGoogleEmail; else the legacy IsOAuth=true row.
            var googleTarget = ResolveIsGoogleTargetRow(user, legacyGoogleEmail);
            if (googleTarget is not null && !googleTarget.IsGoogle)
            {
                foreach (var sibling in user.UserEmails.Where(e => e.Id != googleTarget.Id && e.IsGoogle))
                {
                    sibling.IsGoogle = false;
                    sibling.UpdatedAt = now;
                }

                googleTarget.IsGoogle = true;
                googleTarget.UpdatedAt = now;
                isGoogleRowsUpdated++;

                await _auditLog.WriteAsync(/* user.Id, "UserEmail.Backfill.IsGoogle", ... */);
            }
        }

        await _dbContext.SaveChangesAsync(cancellationToken);
        return new UserEmailProviderBackfillResult(
            usersProcessed, providerRowsUpdated, isGoogleRowsUpdated, ambiguousMatchesWarned, warnings);
    }

    private UserEmail? ResolveProviderTargetRow(
        User user,
        string? legacyGoogleEmail,
        List<IdentityUserLogin<Guid>> allLogins,
        IdentityUserLogin<Guid> login,
        List<string> warnings,
        ref int ambiguousMatchesWarned)
    {
        if (allLogins.Count > 1 && user.UserEmails.Count == 1)
        {
            warnings.Add($"User {user.Id}: {allLogins.Count} AspNetUserLogins rows but only 1 UserEmail; first match wins.");
            ambiguousMatchesWarned++;
        }

        // Resolution order: legacy IsOAuth=true row matching legacyGoogleEmail → row matching User.Email override → first by alphabetical Email.
        var byLegacyOAuthAndEmail = user.UserEmails.FirstOrDefault(e =>
            _dbContext.Entry(e).Property<bool>("IsOAuth").CurrentValue
            && legacyGoogleEmail is not null
            && string.Equals(e.Email, legacyGoogleEmail, StringComparison.OrdinalIgnoreCase));
        if (byLegacyOAuthAndEmail is not null) return byLegacyOAuthAndEmail;

        var byUserEmail = user.UserEmails.FirstOrDefault(e =>
            string.Equals(e.Email, user.Email, StringComparison.OrdinalIgnoreCase));
        if (byUserEmail is not null) return byUserEmail;

        return user.UserEmails.OrderBy(e => e.Email, StringComparer.OrdinalIgnoreCase).FirstOrDefault();
    }

    private UserEmail? ResolveIsGoogleTargetRow(User user, string? legacyGoogleEmail)
    {
        if (legacyGoogleEmail is not null)
        {
            var byEmail = user.UserEmails.FirstOrDefault(e =>
                string.Equals(e.Email, legacyGoogleEmail, StringComparison.OrdinalIgnoreCase));
            if (byEmail is not null) return byEmail;
        }

        return user.UserEmails.FirstOrDefault(e =>
            _dbContext.Entry(e).Property<bool>("IsOAuth").CurrentValue);
    }
}
```
